# Agent 长期记忆中的「记忆一致性」问题

> **English version:** [memory-consistency.md](./memory-consistency.md)
>
> **状态:** 问题定义。本文只**定义问题**,**不规定解决方案**,用于引出未来的 `03` 例子。
>
> **背景:** 例子 [`02-self-evolving-memory`](../02-self-evolving-memory) 给 agent
> 加了「持久、需用户批准」的长期记忆,并在每次新会话时注入。该例子**有意停在最朴素的
> 设计**——四个分类、每个分类一段「完整覆盖式散文」、整体注入。它能**记住**和**使用**
> 事实,却无法在某个前提改变时**修正自己的信念**。本文精确地定义这个缺口。

---

## 1. 问题陈述

一个 agent 的长期记忆**不是**一组相互独立的事实。它是一张由事实构成的网,其中隐含着
**依赖关系**、**有效期**和**出处**。当一条事实改变时,所有**依赖它**的事实都应当被
修正、作废或重新推导。

一个只会**新增**和**检索**、却从不**对账(reconcile)**的记忆系统,会随时间不断
累积**过期的、相互矛盾的**信念。随着记忆变大,「某条已存信念与另一条(或与现实)
相互矛盾」的概率趋近于 1。

我们把这称为**记忆一致性(memory consistency)问题**(它与经典的**信念修正
belief revision** 与**真值维护 truth maintenance** 密切相关)。

核心的不对称在于:大多数记忆系统把**「记住」**当作一等操作,把**「遗忘 / 作废」**
当作事后补丁(或当成「必须由用户显式提出」的事)。而现实中,作废往往是**沉默且需要
推断**的——用户从不会说「忘掉那条」,他们只是陈述一条新事实,而这条新事实的后果会
向外层层波及。

---

## 2. 贯穿示例

| 时刻 | 用户说 | 此后记忆**应当**是什么 |
| ---- | ----- | -------------------- |
| T0 | 「我住在新加坡,走路 15 分钟到公司。」 | `home = 新加坡`;`commute = 走路15分钟`**(由** home + 公司位置**推导而来)** |
| T1(数周后) | 「我搬到西雅图了。」 | `home = 西雅图`;`commute = 走路15分钟` 现在**未经证实 / 很可能失效**,必须被修正或丢弃 |
| T2 | (一个新会话;agent 回答问题) | agent **绝不应**仍然相信用户走路 15 分钟上班 |

**症结所在:** 在 T1,用户只改了一条事实(`home`),对通勤只字未提。一个正确的系统
必须**推断出** `commute` *依赖于* `home`,因此已经过期。`02` 里的扁平散文记忆**根本
没有表示这种依赖**,于是那条过期事实通常会幸存下来——agent 最终既「相信用户住在
西雅图」,又「记得用户走路 15 分钟到公司」。

这个例子仅是一个**具象的引子**。同样的结构无处不在(见 §3)。

---

## 3. 失效模式分类

下面所有情况都共享同一种结构:**某条已存信念本应作为新信息的后果而改变,但系统里
没有任何东西强制这个后果发生。**

| # | 失效模式 | 具体例子 | 正确的系统应当 |
| - | ------- | ------- | ------------- |
| F1 | **派生事实过期** | 住址变了;通勤/邻居/时区/本地习惯都是从住址推导出来的 | 作废或重新推导每一条派生事实 |
| F2 | **直接矛盾** | 「我吃素」→ 后来「我开始吃鱼了」 | 检测冲突;替换,而非追加 |
| F3 | **时效过期** | 「我女儿 3 岁」/「我在 Acme 上班」 | 视为「截至某时有效」,而非永真;随时间老化或重新确认 |
| F4 | **实体变更级联** | 「我的经理是张三」→ 张三离职 | 对每条相关记忆逐一决策:作废、转移或归档 |
| F5 | **重复 / 碎片化** | 同一条偏好在多个会话里被反复陈述 | 合并为唯一的规范条目;不要无限增长 |
| F6 | **沉默撤回** | 用户只是不再提某个项目;或陈述了一条暗示旧事已结束的新事实 | 从上下文**推断**这次撤回;用户很少会说「忘了 X」 |
| F7 | **粒度 / 范围漂移** | 「我喜欢简洁的回答」(通用)vs.「这份报告请详细些」(局部) | 别把一次性要求提升为长期信念,反之亦然 |

F1、F4、F6 最难,因为**触发是隐式的**:用户的新陈述**并没有点名**那条必须改变的事实。

## 4. 根因:扁平散文记忆为何会失败

`02` 的设计是:每个分类存一段自然语言散文——也就是只存了**结论**,而**前提以及它们
之间的关联被丢弃了**。仅这一个选择,就导致三个结构性失败:

1. **无矛盾检测。** 没有可供比对的结构化表示,系统无法察觉一条新事实与某条已存事实
   冲突。检测完全被甩给「模型重新通读一遍」。
2. **无级联。** 因为依赖关系(`commute` *依赖* `home`)没有被记录,系统无法把一个前提
   的改变传播到它的派生项。它根本不知道该回头去看哪些条目。
3. **没有「遗忘」机制。** 「作废」不是一个操作;唯一的写入是「替换某分类的散文」。一条
   过期事实会不会消失,取决于模型在重写时**碰巧**选择把它略去。

更糟的是,`02` 的指令告诉模型写入「*已有内容**加上**新细节*」。这个措辞**偏向累积**
——它在暗示模型**保留**旧的通勤说法,而这恰恰与 F1 的需求相反。

更深一层:**这无法靠一条更聪明的单次 prompt 解决。** 指望 LLM 在每次写入时重新通读
全部记忆、并推理一致性,会随记忆增长而退化——事实越多、可能的相互作用越多、漏掉某条
的机会也越多。一致性需要的是一套**机制**,而不只是更好的指令。

## 5. 一个一致性记忆系统**必须具备**的能力

以下是**问题本身所要求的能力**,以「需求」形式陈述(而非某种设计)。任何解决方案都
会以某种形式满足它们。

- **C1 — 出处(Provenance)。** 每条信念记录它来自哪里(哪句用户陈述、哪个会话、何时),
  以及在相关时,它**派生自**什么。
- **C2 — 依赖 / 推导链接。** 派生信念指向它所依赖的信念,这样前提的改变才能找到它的
  派生项。
- **C3 — 时效有效性。** 信念是「截至某时有效」,而非永真。有些还带有「会过期 / 需
  重新确认」的预期。
- **C4 — 矛盾检测。** 面对新输入时,系统能找出与之冲突或被其削弱的已存信念。
- **C5 — 级联作废 / 重新推导。** 改变一条信念会触发对其派生项的一次遍历,去更新、
  作废或重新推导它们。
- **C6 — 去重 / 合并。** 重复或重叠的陈述收敛为一条规范信念,而不是不断堆积。
- **C7 — 把「遗忘」做成一等操作。** 「作废 / 撤回 / 归档」是显式操作,最好可审计
  (丢了什么、为什么),而不是重写散文时的副作用。
- **C8 — 置信度 / 状态。** 一条信念可以是*已确认*、*推断得来*、*疑似过期*、*已撤回*
  ——而不只是「存在 / 不存在」。

### 对账(reconciliation)步骤

上面这些能力,隐含着一个 `02` 完全缺失的操作。称之为**对账(reconciliation)**:
给定 *(当前记忆,新信息)*,产出一份**自洽的后继记忆**,外加一份**变更日志**——记录
新增了什么、更新了什么、作废了什么、合并了什么、以及哪些被标记为需重新确认。**对账**
——而不是原始的「追加一条事实」——才是一致性记忆系统真正的写入路径。它**在哪里运行**
(写入时 / 后台批处理 / 读取时)、**变更日志是否需要人工批准**,都属于本文**留待开放**
的**设计**问题。

## 6. 范围与非目标

**在范围内。** 单用户的*个人*长期记忆:关于用户及其世界的、跨会话的持久信念(身份、
偏好、处境、人际关系、项目)。

**不在范围内(就本问题定义而言)。**

- **情节 / 逐字记忆(episodic / transcript)**——逐字回放过去对话。本文讨论的是
  *语义信念*,不是日志。
- **文档 RAG**——对外部语料的检索。那是检索问题;本文是*信念一致性*问题。
- **多用户 / 共享记忆**及跨用户的冲突消解。
- **隐私、PII、被遗忘权**这类合规问题。真实存在,但与逻辑一致性是不同的维度。
- **规定数据模型或算法。** 本文有意不予定义。

## 7. 如何判定一个方案是否有效(评测场景)

评判一个方案,应当看**记忆的状态迁移**,而不只看聊天回复。下面每个场景都是一串输入
及其每步之后**期望的记忆状态**——它们将成为未来 `03` 例子的 eval。

1. **搬家(F1)。** 「新加坡,走路 15 分钟」→「搬到西雅图」。*期望:* 住址更新;通勤
   被作废或标记为疑似过期;不残留「走路 15 分钟」。
2. **饮食反转(F2)。** 「吃素」→「开始吃鱼」。*期望:* 一条自洽的饮食信念;不存在
   相互矛盾的一对。
3. **换工作级联(F4)。** 「在 Acme 上班,经理张三,通勤到市中心」→「离开了 Acme」。
   *期望:* 雇主 / 经理 / 通勤都被重新审视,而不只是雇主。
4. **重复(F5)。** 同一条偏好在三个会话里被陈述。*期望:* 单条规范条目,而非三条。
5. **沉默丢弃(F6)。** 某活跃项目被提及数周,之后一条新陈述暗示它已上线/结束。
   *期望:* 旧项目被归档/关闭,而无需用户说「忘了它」。

一个合格的方案,在每一步之后都让记忆保持**内部自洽且与当前一致**,并留下一份**「改了
什么、为什么改」**的可审计记录。

## 8. 术语与相关工作

- **信念修正(Belief revision)**——研究一个理性 agent 在面对新的、可能冲突的信息时
  应当如何改变其信念(经典框架为 *AGM*)。本文的问题是它的一个应用实例。
- **真值维护系统(TMS / JTMS)**——一种经典 AI 机制,记录信念的*正当理由
  (justification)*,并在某个理由失效时撤回其派生项。直接对应 C2 + C5。
- **时态 / 双时态知识(temporal / bitemporal)**——把事实建模为在时间窗口内有效,
  而非永恒为真(C3)。
- **LLM 知识编辑(knowledge editing)**——在不完全重训的前提下更新模型存储的知识;
  相关,但它针对的是*模型权重*,而本文针对的是一个*外部、可检视的记忆库*。
- **记忆固化 / 对账(memory consolidation / reconciliation)**——agent 记忆领域里
  (尚不太标准化的)术语,指周期性地压缩并消解所累积记忆中的冲突。
